Fastest reading order

Start with the lifecycle, then layer in self-serve assists

The primary lifecycle is owner draft review in quincy, governance review, and then either publication or a targeted return to owner. The policy bot and ruleset QC mockups are support paths that help users prepare faster before they ever enter the governance cycle.

Primary lifecycle

The main end-to-end path

This is the core process a document owner and governance team would follow for an annual refresh or proposed policy change.
Step 1

Owner prepares the draft in quincy

The owner starts from the LogicGate-linked working Google Doc, explains what changed, runs quincy QC, and prepares the owner package with a change summary and QC audit trail.

Primary entry Google Doc remains source
  • Capture the delta
  • Run QC against the draft
  • Clear must-fix items
  • Export the submission package
Open annual refresh mockup
Step 2

Governance receives and reviews the package

The governance workspace imports the owner package, brings the next record forward, prepares packet summaries and return notes, and decides whether the record moves ahead or comes back with targeted fixes.

Decision point System of record remains LogicGate
  • Review owner package and QC context
  • Prepare packet summary or return note
  • Move the record forward or send it back
Open governance mockup
Step 3

Record is approved and continues the normal lifecycle

If governance is satisfied, the package moves into approval and publication without extra owner cleanup. The approved document remains governed in the existing LogicGate and Google Drive lifecycle.

Forward path No extra mockup required
  • Approval routing continues
  • Publication and record updates complete
  • Next review cycle resumes normally

Return loop

If governance sends the package back

This is the targeted rework loop: only the returned items come back to the owner, and the preserved package avoids rebuilding work that governance already accepted.
Governance outcome

Returned with required fixes

Governance identifies the exact missing items and prepares a note so the owner knows what to change next and what can stay preserved.
